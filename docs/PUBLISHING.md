# Publishing

This repository publishes preview VSIX files through GitHub Releases. Marketplace publishing is manual until the first public upload has been verified.

## Local Preflight

```bash
pnpm install --frozen-lockfile
pnpm verify:release
```

The release gate runs extension tests, VS Code host smoke tests, VSIX packaging, VSIX hygiene checks, VSIX content verification, and packaged install smoke tests.

## GitHub Release

1. Update `extension/package.json` and `extension/CHANGELOG.md`.
2. Commit the release changes.
3. Push `main`.
4. Create and push a tag that exactly matches the extension version:

```bash
git tag v0.5.0
git push origin v0.5.0
```

The `Release VSIX` workflow verifies the tag against `extension/package.json`, runs `pnpm verify:release`, and uploads `extension/guruguru-codechan-<version>.vsix` to a GitHub pre-release.

## First Marketplace Upload

Manual steps:

1. Download the VSIX from the GitHub Release.
2. Confirm the Marketplace README, icon, preview image, license text, and `ASSET_LICENSE.md`.
3. Upload the VSIX with publisher `hitsuki-ban`.
4. Keep the release marked as a pre-release while the extension is still in the 0.x line.

## Future Marketplace Automation

After the first public Marketplace upload is verified:

1. Add `VSCE_PAT` as a repository secret.
2. Add a release workflow step that runs `pnpm --filter guruguru-codechan exec vsce publish --packagePath guruguru-codechan-<version>.vsix`.
3. Keep the GitHub Release VSIX artifact as the canonical package generated by CI.
